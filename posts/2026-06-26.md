What I adopted and what I deliberately ignored this quarter — building marketing on open source.

Adopted:

→ Firecrawl — turned the live web into a data source. Now the default for any research job.

→ browser-use — closed the "no API" gap. If a human can do it in a browser, the agent can.

→ MCP servers — stopped writing custom connectors. The stack now plugs into any model the same way.

Skipped, on purpose:

→ n8n and visual workflow builders — great for simple glue, but the moment logic gets real you're debugging boxes and arrows instead of code. I'd rather write 50 lines I can read than wire 30 nodes I can't.

→ "All-in-one AI marketing platforms" — they sell you the wiring as a subscription. The wiring is the part you should own.

→ Most agent frameworks with ten abstractions — for one operator, a plain loop and good prompts beat a framework you have to learn.

The pattern in what I keep: primitives I can compose. The pattern in what I drop: abstractions that hide the system from me.

You don't need more tools. You need fewer, more composable ones — and to own the layer that connects them.

That layer is the whole job.
